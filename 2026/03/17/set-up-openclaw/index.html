零成本安装配置OpenClaw，但OpenClaw真的有用吗？
irmenth Lv1
  2026-03-17 20:40:44   2026-03-17 20:48:15  1.8k 字  

前言

相信大家最近有经常听到一个词：龙虾🦞，也就是我们今天的主角：OpenClaw。 要了解OpenClaw，我们首先要澄清一个误区，就是OpenClaw并不是类似qwen3.5系列或者gpt系列的大语言模型（LLM），它是一个写死的程序，并没有类似LLM那样的智能。 OpenClaw是一个AI Agent，对比之前的Agent，它更加轻量化，同时不再像之前的Agent那样依赖云端，反而会把各种记忆和数据存储在本地。 他的爆火或许说明AI Agent已经逐渐趋于成熟。

但同时，大家也需要明白，AI Agent并不是像现在炒作的那样，你现在不用就会被淘汰。 在博主看来，现在的AI Agent的使用性价比或许是远低于直接在各家LLM官网去免费试用他们的LLM的。 因为刚才也说过了，AI Agent并不是人工智能，它需要连接一个LLM来作为他的大脑，单独的Agent基本上什么都做不了。

但各大公司首先并不一定会开源他们官网上最先进的LLM；其次，开源的大参数的LLM（16B往上）在普通人的电脑上基本跑不起来，就比如说qwen3.5的开源系列，一般大家的显存可能大概是8G上下，这个大小大概只能跑得动9B参数的LLM（或许勉强还可以往上一点，但更多参数的LLM代表响应速度会更慢）。 但很遗憾，如果我们想把9B参数大小的LLM连上OpenClaw进行使用，那大概率还是行不通的。 刚才不是说可以勉强跑得动吗，现在怎么用不能用了呢？ 因为OpenClaw这种AI Agent对于LLM的使用和我们直接跟LLM聊天是不一样的。 我们输入“你好”，LLM得到的prompt最多是“用户说：你好。你回答：”，或者加上之前的聊天记录“之前的聊天记录…用户说：你好。你回答：”，然后LLM就会开始文字接龙，最后我们会得到回答。 但OpenClaw会把他的配置文件里的内容包括用户的输入全部给到LLM，而且我们的一次对话在OpenClaw与LLM之间可能会进行多次。

既然这样，就不能使用云端LLM吗？ 可以，但是要付费（虽然也有被提供免费云端服务模型，但智能程度嘛…）。 而且我们刚才也讲过了，OpenClaw每次给出的prompt里面含的token数量很多，且OpenClaw与我们的一段对话，往往会与LLM进行多次。 所以这个消费在博主看来也是比较夸张的。

这次的前言很长，博主主要是想告诉大家，这个OpenClaw并不是非装不可。 反而博主认为，你如果不是因为想要体验这个火爆的软件（想玩一下），那是真的完全没必要浪费这个时间。 或许在未来，低成本的个人AI Agent真的会成为我们日常生活中不可分割的一部分，但博主认为，至少还不是现在。

好了，如果你有耐心能看到这里，并且确实也想玩一下，那就让我们继续吧。

配置

终端输入

就可以进入OpenClaw的终端交互式配置界面（如果这里的配置选错了也没有关系，我们后续可以在配置文件进行修改）。 我们选择Quick Start。 博主这里的LLM供应商选择OpenRouter，选择其他的大家也可以自行配置，流程都是一样的，进入官网，然后Get API Key，然后填进去就可以了。 如果想用本地的LLM，可以选择Ollama（LLM不需要再去手动选择调教版本了，但Ollama的速度会很慢）或者vLLM。 接着会让我们输入OpenRouter的API Key。 我们来到OpenRouter的官网，点击Get API Key，然后创建一个复制过来。 模型的话各位就自行挑选了，可以在OpenRouter里面搜索free，然后进行选择。

接下来配置Channel，如果各位只需要在网页进行对话那这一步可以跳过。 这里博主使用Telegram进行演示。 我们选择Telegram，然后需要输入Bot API Token。 我们打开Telegram，搜索BotFather（或者点这个网址）。 输入/new根据指引创建一个新的机器人。 创建完成以后复制他的API Token过来。

接下来配置搜索引擎，博主这里使用Gemini AI搜索来做演示。 首先我们到Gemini API Key去创建一个新的API Key（使用默认的也可以），然后复制过来。
